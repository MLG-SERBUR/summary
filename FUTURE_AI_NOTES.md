# Future AI Notes

Use this repo as the human-readable memory for the server resurrection work.

## Start Here

1. Run `scripts/collect-context.sh <repo...>`.
2. Check the server repo and plugin clones for fresh commits and uncommitted changes.
3. Update `PROGRESS_REPORT.md` with the newest date, wins, blockers, and any notable plugin work.
4. If the server worktree is still messy, say that plainly instead of hiding it.
5. Keep `VISUAL_REPORT.html` numbers and wording in sync with newest published summary.
6. Keep `STORY_REPORT.html` diary entries aligned with latest repo highlights.

## Rules

- Do not commit player IPs.
- Treat YAML key-order churn as noise unless the content actually changed.
- Prefer small progress updates over giant rewrites.
- If something is generated or disposable, call it out instead of pretending it is source material.
- Keep publishable docs free of the real server-directory path.

## Current Hot Spots

- `MountainDewritoes` and repos in the `MLG-Fortress` Github org are where a lot of the visible upgrade work has landed.

- What changed.
- Why it mattered.
- What is still annoying.
