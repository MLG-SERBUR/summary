Diary entries, in order

Times are approximate and based on the commit trail. Gaps matter here. The work did not happen all at once.
Apr 14

First pass on MountainDewritoes

Early evening, the plugin started losing old baggage. `GrandioseAPI` came out, player data moved local, and the main plugin took over more of the logic.

A few hours later that simplification got revised because the main class was already too crowded. So the player data code came back as a small wrapper around PDC storage. Less grand. More readable.

dependency removal PDC storage main class too big
Apr 15

The build stopped being polite

Late morning, the work shifted to build plumbing. Repository URLs were updated, then shade plugin trouble forced Java back to 21 so the class files would stop fighting the toolchain.

A little later the updater got switched back on. The pattern was clear by then: if the build path is wrong, the rest of the server story cannot move forward.

repo URLs Java 21 updater
Apr 17-18

The org learned how to build itself again

Over that stretch, lots of repos got the same kind of treatment: new GitHub Actions workflows, old workflows removed, Maven quieted down, and modern Paper/Purpur compatibility work spread through the stack.

This is where the work stopped feeling like one plugin rescue and started feeling like the whole ecosystem was being pulled forward together.

CI cleanup Paper 26.1.2 workflow churn
Apr 19

PrettySimpleShop finally told the truth about items

Afternoon into evening. First the item-name fallback chain got reordered. Then translated names were made consistent. Then the shop code got the actual shop item name instead of the raw itemstack name.

That sounds tiny. It was not tiny. It was the kind of bug that only matters when the whole feature is wrong in a subtle way.

item-name fallback translated names shop behavior
Apr 20

ItemNameLab turned into useful tooling

Morning. New test plugin. Then command arguments shifted. Then output became components. Then tests were added for custom and translated names. That sequence matters because it shows the work maturing while it moved.

The point was not to build a huge tool. The point was to make item-name behavior easier to trust everywhere else.

tests components translated names
Apr 20

MountainDewritoes had to choose between neat and real

Later that same day, the PDC split got reconsidered. The main class was getting huge, so the wrapper came back. Then startup and config compatibility got trimmed down because the plugin had to load cleanly before anything else mattered.

That kind of reversal is normal here. Sometimes the first clean design is not the best one once the code gets real.

PDC wrapper startup fixes less duplication
Apr 21

CreativeParkour and the rest of the stack started surviving the new world

The build side tightened up again: old workflows got removed, batch Maven got a quieter flag, and the server stack kept moving toward modern API compatibility.

By evening, a small NPE fix in MountainDewritoes showed the next stage of work. Once the big compatibility gaps are gone, the little runtime bugs finally come into view.

workflow cleanup NPE fix runtime polish
Apr 22

Server ping and shop fixes landed near the end

Evening. PrettySimpleShop got one last correction so it would use the actual shop item name.

In the same stretch, MountainDewritoes dropped ServerListPlus and took over the server ping path directly with its own listener. Fewer layers. Less mystery. More control.

shop item name server ping drop ServerListPlus
